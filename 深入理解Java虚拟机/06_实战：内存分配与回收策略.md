#  实战：内存分配与回收策略

对象的内存分配，从概念上讲，应该都是在***堆上分配***（而实际上也有可能经过即时编译后被拆散为标量类型并间接地在栈上分配[插图]）。在经典分代的设计下，***新生对象***通常会分配在***新生代***中，***少数***情况下（例如对象大小超过一定阈值）也可能会直接分配在***老年代***。对象分配的规则取决于虚拟机使用***垃圾收集器***，以及虚拟机中与内存相关的***参数***的设定。



### 对象优先在Eden分配

大多数情况下，对象在新生代***Eden***区中分配。当Eden区没有足够空间进行分配时，虚拟机将发起一次***Minor GC***。



```java
/**
 *-Xms20M 最小堆空间
 *-Xmx20M 最大堆空间
 *-Xmn10M 新生代空间
 *-XX:+PrintGCDetails 
 *-XX:SurvivorRatio=8 Eden:From Survivor: To Survivor 8:1:1
 */
public class LocalTest {
    private static final int _1MB = 1024 * 1024;
    public static void main(String[] args) {
        byte[] allocation1, allocation2, allocation3, allocation4;
        allocation1 = new byte[2 * _1MB];
        allocation2 = new byte[2 * _1MB];
        allocation3 = new byte[2 * _1MB];
        allocation4 = new byte[4 * _1MB];  //出现一次Minor GC
    }
}
```
